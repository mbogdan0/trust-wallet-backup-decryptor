# Security Policy

## Reporting a Vulnerability

Please report security issues privately by email: **windbogd@gmail.com**.

When sending a report, include:

- A clear description of the issue and impact.
- Steps to reproduce (or a minimal proof of concept).
- Affected commit/tag and environment details.

I will acknowledge receipt as soon as possible and coordinate a responsible fix/disclosure timeline.

## In Scope

- Cryptographic correctness and misuse-resistance in decryption flow.
- Offline invariant enforcement (no network/data exfiltration behavior).
- Build artifact reproducibility and integrity checks.

## Out of Scope

- Compromise of the user's machine, browser, OS, extensions, or local environment.
